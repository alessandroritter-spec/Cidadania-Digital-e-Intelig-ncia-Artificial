# Cidadania Digital e Inteligência Artificial: O Impacto das Deepfakes e da Desinformação na Sociedade 🌐🤖

Este portal web interativo foi construído para disseminar a conscientização pública sobre os perigos da desinformação algorítmica e deepfakes na era da Inteligência Artificial Generativa.

## 🛠️ Tecnologias Utilizadas
* **HTML5:** Estruturação semântica adaptada a leitores de tela.
* **CSS3:** Layout estruturado em Flexbox e responsividade completa via Media Queries.
* **JavaScript (Vanilla):** Mecânicas interativas completas de manipulação do DOM e integrações de API nativas.

## 🌟 Funcionalidades e Critérios Atendidos
1. **Mascote Assistente Dinâmico:** O bot `CyberGuard` interage textualmente modificando balões de diálogos a cada ação ou fase do jogo executada pelo usuário.
2. **Verificador de Links:** Filtragem dinâmica que avisa o usuário se as URLs apresentam padrões textuais comuns em ataques digitais de engenharia social.
3. **Simulador de Ligação por Voz IA:** Usa a API nativa de síntese de voz do navegador (`window.speechSynthesis`) para simular em tempo real um ataque telefônico por clonagem de áudio.
4. **Jogo Educativo por Fases:** Quizzes sequenciais divididos por pontuações e avanço de fases que ensinam condutas baseadas em Cidadania Digital.
5. **Acessibilidade de Contraste:** Botão de controle de tema para alternância dinâmica entre Modo Claro e Escuro.

## 🧠 Prompts Utilizados na Concepção da IA
> "Atue como um desenvolvedor Front-end sênior especialista em acessibilidade. Desenhe uma aplicação semântica em HTML5 puro sem frameworks que aborde segurança de dados contra desinformação gerada por Inteligência Artificial. Construa rotinas nativas em JS para simulação de ligações de voz falsas e verificação de estruturas sintáticas de URLs para estudantes de ensino médio."

## 🧠 Documentação de Prompts de IA (Critério Aula 4 / Subcategoria D)

Para o desenvolvimento da identidade visual, estrutura lógica e banco de dados simulados deste portal, foram utilizados assistentes baseados em Inteligência Artificial Generativa. Abaixo estão documentados os prompts exatos aplicados:

### 1. Arquitetura e Estrutura Semântica (HTML5)
> **Prompt:** "Atue como um desenvolvedor Front-End Sênior focado em acessibilidade web. Escreva o código estrutural de um arquivo index.html usando tags semânticas do HTML5 (main, section, header, footer, form, label). O tema do site deve ser 'Cidadania Digital e IA: O Impacto das Deepfakes e Desinformação'. O site precisa conter um cabeçalho com botão de acessibilidade, uma seção de boas-vindas com um mascote bot e três seções de ferramentas interativas: um validador de links, um jogo em fases e uma central de denúncias."

### 2. Estilização Moderna e Design Responsivo (CSS3)
> **Prompt:** "Crie um arquivo style.css moderno e limpo com o conceito Mobile-First para um portal de segurança digital. Utilize CSS Variables para suportar a troca de tema (Modo Claro/Escuro usando o atributo data-theme='dark'). Configure os cartões das seções utilizando Flexbox estruturado para que, em telas menores de 600px, os elementos como botões paralelos e caixas de imagem fiquem em colunas verticais automáticas. Adicione animações de pulse para o avatar do robô mascote e efeitos hover suaves nos botões."

### 3. Mecânicas Avançadas e Manipulação do DOM (JavaScript)
> **Prompt:** "Desenvolva o arquivo script.js para um portal interativo usando JavaScript puro (Vanilla). Crie uma matriz de objetos com 3 perguntas complexas sobre Deepfakes e Cidadania Digital contendo alternativas corretas e incorretas. Programe funções que alterem dinamicamente o texto do balão de diálogo de um mascote robô na tela dependendo das interações do usuário. Escreva a lógica para fazer o navegador falar um texto simulado de golpe de clonagem de voz usando a API nativa 'window.speechSynthesis' de forma assíncrona."

### 4. Engenharia Social e Validação de Dados (Segurança)
> **Prompt:** "Escreva regras simples de validação de string em JavaScript para simular o comportamento de uma extensão de segurança de navegador. O script deve receber uma URL digitada pelo usuário e procurar por palavras-chave comumente associadas a golpes de phishing (como 'ganhe-gratis', 'urgente') e checar se o protocolo inicial começa obrigatoriamente com 'https://', exibindo alertas estilizados na tela com base nessa análise estrutural."
